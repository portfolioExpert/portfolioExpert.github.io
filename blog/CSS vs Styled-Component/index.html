CSS와 Styled-component

CSS와 Styled-component의 차이점을 정리하였습니다.

SCSS vs Styled-Component

참고: https://velog.io/@gur0601/Styled-components%EC%99%80-SCSS-%EC%B0%A8%EC%9D%B4

이번에 사이드 프로젝트를 하면서 styled-component를 사용해봤는데 기존에 사용했던 scss와 어떤 차이가 있고 왜 사용하는지 궁금하여 정리하게 되었습니다.

Scss

  • - 코드의 재활용성을 올리고, 가독성을 올리는 등 CSS의 단점을 보완하고 개발의 효율을 올리기 위해 사용한다.

장점

  • - CSS보다 간단한 표기법, CSS를 구조화하여 표현

  • - Nesting, Mixin, selector등의 기능을 활용하여 CSS유지보수 편의성을 향상시킨다.

단점

  • - 전처리기를 위한 도구가 필요

  • - 컴파일 하는데 시간이 너무 오래걸림

  • - class명을 일일이 정해줘야함

특징

  1. 변수 예시

정의하고 가져다 쓸 수 있다

$color: #ed5b8;
$borderRadius: 5px;

div{
    background: $color;
    border-radius: $borderRadius;
}
  1. 중첩

label에 호버 기능을 주는 것 처럼 중첩으로 사용 가능하다.

label {
      padding: 3% 0px;
      width: 100%;
      cursor: pointer;
      color: $colorPoint;

      &:hover {
        color: white;
        background-color: $color;
      }
}
  1. import

다른 scss 파일을 import하여 사용할 수 있다.

// common.scss

// colors
$color1: #ed5b46;
$color2: #f26671;
$color3: #f28585;
$color4: #feac97;


//style.scss
@import "common.scss";

.box {
	background-color: $color3;
}

Styled-Component

  • - 컴포넌트 기반으로 css를 작성할 수 있도록 해주는 라이브러리
const button = styled.button`
    /*공통 스타일*/
    display: inline-flex;
    align-items: center;
    outline: none;
    border: none;
    border-radius: 4px;
    color: white;
    font-weight: bold;
    cursor: pointer;
    padding-left: 1rem;
    padding-right: 1rem;


    /*크기*/
    height: 2.25rem;
    font-size: 1rem;

    /*색상 */
    background: #228be6;
    &:hover{
        background: #339af0;
    }
    &:active{
        background: #1c7ed6;
    }

    /*기타 */
    & + & {
        margin-left: 1rem;
    }
`;

장점

  • - scss 라이브러리 설치 없이 컴포넌트 안에 scss 문법을 사용할 수 있다.

  • - 자유로운 css 커스텀 컴포넌트를 제작할 수 있다.

  • - 컴포넌트의 props나 상태를 참고하여 다르게 스타일을 적용할 수 있다.

단점

  • - 빠른 페이지 로드에 분리하다.

특징

  • - Automatic critical Css: 화면에 컴포넌트가 렌더링 되었는지 추적해 컴포넌트에 대한 스타일을 자동 삽입한다. 코드를 적절하게 분배해 놓으면 최소한의 코드만으로 화면을 띄울 수 있다.

  • - No class name bugs: styled-component는 스스로 className을 생성하기 때문에 className의 중복이나 오타로 인한 버그를 줄여준다.

  • - Easier deletion of CSS: styled Component는 모든 스타일 속성이 특정 컴포넌트와 연결되어 있기 때문에 더 이상 사용하지 않아 삭제할 경우 이에 대한 스타일 속성도 함께 삭제된다.

  • - Simple dynamic styling: className을 일일이 수동으로 관리할 필요 없이 React의 props나 전역 속성을 기반으로 컴포넌트에 스타일 속성을 부여하기 때문에 간단하고 직관적

  • - Painless maintenance: 컴포넌트에 스타일을 상속하는 속성을 찾아 다른 CSS 파일들을 검색하지 않아도 되기 때문에 코드의 크기가 커지더라도 유지보수가 어렵지 않다.

  • - Automatic vendor prefixing: 컴포넌트마다 기존 CSS 문법으로 스타일만 정의하면 될 뿐이다.

SCSS와 Styled-component 차이점

SCSS

  • - css in css 문법으로 js 파일과 분리되어 있다.

  • - 어떤 컴포넌트의 상태값이 변경되더라도 반응하기 쉽지 않다.

  • - 브라우저에 보이지 않은 컴포넌트까지 읽기 때문에 불필요한 컴파일 과정이 추가

styled-component

  • - css in js 방식으로 컴포넌트가 렌더링 될 때만 해당 스타일 정보를 읽는다.

  • - 동적인 이벤트가 많은 사이트라면 컴포넌트가 자주 렌더링 될 때 그만큼 스타일 정보도 다시 읽어 와야 한다.
