Sass와 SCSS

간단한 Sass와 SCSS에 대해서

Sass, SCSS

Sass나 SCSS 쓰는 이유

  • - Sass안에 변수, 네이스팅, 믹스인, 가져오기, 상속, 내장기능 등 css에는 없는 편의 기능을 제공해 시간을 절약할 수 있다.

  • - 코드 재사용이 가능하다.

Sass 란

  • - 스타일 시트 엔진: 특정한 형태의 스타일시트를 CSS 스타일 시트로 변경해주는 변환 엔진 컴파일러

  • - 브라우저가 이해할 수 있는 CSS로 변환해주는 도구

  • - 다시 CSS로 컴파일해줄 프로세서

  • - 확장자는 .Sass, .SCSS

파일분리와 Nesting

  • - 각 프레임 별(_header.scss, _home.scss) css를 분리하고, variable과 mixing 파일도 따로 분리

  • - 메인 파일인 style.scss에 _파일들을 import 해주며 메인으로 실행

파일명에 언더바 _를 붙이는 이유

_를 붙이지 않는다면 분할 된 파일들 모두가 컴파일되기 때문에 여러 개의 .css파일이 나눠서 저장됩니다. 그러나 scssvㅏ일의 이름 앞에 _를 붙여서 파일명을 정한다면, Sass에게 이 파일이 main파일의 일부분임을 알려줘 해당 파일은 css파일로 컴파일 하지 않고 내부에서 import 형태로 작동하게 됩니다.

  • - Nesting을 사용하면, html의 시각적 계층 방식과 동일하게 CSS를 중첩하여 작성할 수 있습니다. CSS코드가 구조화 되어 가독성이 높아지며 유지 보수가 편리합니다.

Ex)

nav{
  background: lightcoral;
  ul{
    display: flex;
    justify-content: flex-end;
    list-style: none;
    li{
      color: white;
      margin-right: 10px;
    }
  }
}

중첩을 사용하는 이유

기존 CSS는 부모에게 상속된 자식 요소에게 스타일을 적용하려고 할 때마다 최상위 선택자를 반복 선언해야 합니다. 하지만 중첩을 사용하면 최상위 선택자를 한 번만 선언하여도 자식 선택자에게 스타일을 적용할 수 있게 되어 코드 반복을 줄이게 됩니다.

ampersand 앰퍼샌드
  • - '&'는 상위에 있는 부모선택자를 가리킵니다.
@at-root
  • - @at-root 키워드를 사용하면 중첩에서 벗어날 수 있습니다. 중첩에서 벗어나고 싶은 선택자 앞에 작성
CSS 변수
  • - 값을 일일이 고치지 않아도 되는 의미이며 유지보수를 쉽게 만들어 줍니다.

단, 변수를 사용하는 기준이 필요

  • - 값이 두 번이상 반복되면 미리 변수로 만들어두기
  • - 기존의 값을 다른 값으로 변경해야할 경우, 변수의 값만 변경하면 되니 수정될 가능서이 있는지 고려해야한다.

ex)

$bgColor: red;

h1{
  color: $bgColor;
}
변수 type
  • - numbers, strings, color, booleans, lists, null, maps

ex)

$font-weight: ("regular": 400, "medium": 500, "bold": 700);
//맵 사용 방법
map.get($font-weights, "medium");

map-get(map, key)//키에 해당하는 값을 리턴하는 함수
map-keys(map)//map에 들어있는 키를 전부 리턴하는 함수
map-values(map)//map에 들어있는 값(value) 전부를 리턴하는 함수


//리스트
$valid-sides: left, center, right;
p{
  text-align: nth($valid-sides, 1);
}
  • - 리스트는 순서가 있는 값으로, 값마다 인덱스를 가지고 있으며 인덱스는 1부터 시작.
변수의 Scope
  • - 지역변수: {} 안에서 선언한 변수
Mixin
  • - 선택자들 사이에서 반복되고 있는 코드들은 mixin을 사용하여 코드 반복을 줄입니다.

Ex)

.card{
  display: flex;
  justify-content: center;
  align-itmes: center;
  background: gray;
}

.aside{
  display: flex;
  justify-content: center;
  align-items: center;
  backgroud: white;
}

//일 때 mixin으로 변경
@mixin center-xy{
  display: flex;
  justify-content: center;
  align-itmes: center;
}

.card{
  @include center-xy;
}

.aside{
  @include center-xy;
}
Extend

  • - 연관 있는 요소들끼리 스타일 코드가 중복된 경우에 사용.

  • - mixin은 관계없는 선택자에서 조금 다른 스타일을 적용할 때 사용.

  • - extend는 관계있는 선택자들의 동일한 소스코드 적용시 사용

Ex)

.profile-user{
  background-image: url("./programmers.png");
  background-size: cover;
  background-position: 50% 50%;
  border-radius: 50%;
  width: 50px;
  height: 50px;
}

//.profile-user를 extend로 가져오겠다.
.comment-user{
  @extend .profile-user;
}

  $result: 1;
  @for $_from 1 through $exponent{
    $result: $result * $base;
  }
  @return $result;
}

  margin-left: pow(4,3) * 1px;
}
